## 主题安装

Hugo 主题安装有多种方式，以下是几种常见的方法：

### 方法一：使用 Git 子模块（推荐）

使用 Git 子模块是管理 Hugo 主题的最佳方式，它可以让你轻松跟踪主题更新。

```shell
git submodule add <repo-url> themes/<theme-name>
```

例如，安装 PaperMod 主题：

```shell
git submodule add https://github.com/adityatelange/hugo-PaperMod.git themes/PaperMod
```

### 方法二：直接克隆主题

如果不使用 Git 子模块，可以直接克隆主题到 themes 目录：

```shell
git clone <repo-url> themes/<theme-name>
```

### 方法三：下载主题压缩包

1. 从主题的 GitHub 页面下载 ZIP 压缩包
2. 解压到 `themes/<theme-name>` 目录

### 配置主题

安装主题后，需要在 Hugo 配置文件中启用它。在 `hugo.toml` 中添加：

```toml
theme = '<theme-name>'
```

### 初始化子模块

如果是新添加的子模块，需要初始化和更新：

```shell
git submodule init
git submodule update
```

或者使用一条命令：

```shell
git submodule update --init --recursive
```

## 主题卸载

卸载 Hugo 主题的方法取决于你最初是如何安装它的。

### 卸载 Git 子模块主题

如果主题是作为 Git 子模块安装的，使用以下命令：

```shell
git submodule deinit themes/<theme-name>
git rm -f themes/<theme-name>
```

然后提交更改：

```shell
git commit -m "Remove theme <theme-name>"
```

### 清理 .git/modules 目录

有时候，即使执行了上述命令，主题的 Git 数据可能仍保留在 `.git/modules` 目录中。可以手动清理：

```shell
rm -rf .git/modules/themes/<theme-name>
```

### 卸载直接克隆的主题

如果主题是直接克隆的（非子模块），只需删除主题目录：

```shell
rm -rf themes/<theme-name>
```

### 更新配置

卸载主题后，记得更新 Hugo 配置文件 `hugo.toml`，删除或修改 `theme` 配置项。

### 检查卸载是否成功

使用以下命令检查子模块列表，确认主题已被移除：

```shell
git submodule status
```
